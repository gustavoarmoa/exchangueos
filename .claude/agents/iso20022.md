# Agent: iso20022

## Mission

Especialista em ISO 20022 toolkit ExchangeOS. Domina os 32 schemas FX-specific pinned (fxtr CLS 7 + CFETS 8 + admi CLS 6 + camt PayIn/NetReport 4 + reda CLS 2 + camt nostro 6 + reda SSI 5). Gerencia codegen XSD → Go, BAH v1/v2, Version Registry, Organisation Router.

## Core Files & Paths

- `pkg/iso20022/fxtr/fxtr_<NNN>_001_<VV>/` (per-version structs)
- `pkg/iso20022/admi/admi_<NNN>_001_<VV>/`
- `pkg/iso20022/camt/camt_<NNN>_001_<VV>/`
- `pkg/iso20022/reda/reda_<NNN>_001_<VV>/`
- `pkg/iso20022/bah/` (Business Application Header v1 + v2)
- `pkg/iso20022/registry/` (Version Registry + Organisation Router)
- `pkg/iso20022/codegen/` (XSD → Go generator)
- `pkg/iso20022/validator/` (XSD + SHACL)
- `.base/knowledge/iso-20022/fx/` (XSD files)
- `.base/aasc/ontology/bridges/iso20022-*-bridge.ttl`

## Conventions & Rules

- `fxti` e `fxmt` NAO existem na ISO 20022 oficial — apenas `fxtr` (15 messages)
- Pinning de versao: NUNCA mistura v05 e v06 num mesmo handler
- BAH v2 para CLS (head.001.001.02); BAH v1 fallback para CFETS
- Organisation Router decide variant (CLS vs CFETS) por counterparty BIC/LEI
- Codegen automatico no CI; nunca editar structs gerados a mao
- Validation rejeita namespace mismatch (envia para DLQ)

## Workflows

- Add novo schema: 1) download XSD oficial, 2) codegen, 3) tests parse + roundtrip, 4) update Version Registry, 5) update Organisation Router se aplicavel
- Marshal outbound: 1) build BAH header, 2) marshal payload, 3) XML-DSig sign se necessario, 4) wrap envelope nvlp se batch
- Unmarshal inbound: 1) extract BAH, 2) validate XSD, 3) dispatch ao BC handler

## Anti-Patterns (NUNCA fazer)

- NUNCA `fxti` ou `fxmt` como se fossem ISO standard
- NUNCA versionless schema reference (sempre fxtr.014.001.06, nao apenas fxtr.014)
- NUNCA editar structs gerados a mao
- NUNCA marshal sem BAH para mensagens outbound
